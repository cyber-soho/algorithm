Algorithm Strategies: Brute-Force

"When in doubt, use brute force."
— Ken Thompson, co-creator of Unix

What is Brute-Force?

A brute-force algorithm is a straightforward strategy that systematically checks every possible candidate solution until a satisfactory one is found. It relies on sheer computing power rather than clever optimization.

  • ✓
    Complete: Examines every single candidate. No shortcuts.
  • ✓
    Correct: Guaranteed to find a valid answer if one exists in the search space.
  • ✓
    Simple: Trivial to implement, serving as a baseline for complex algorithms.

The Scaling Problem (Big-O Notation)

The fundamental limitation of brute-force is how the cost (time/computation) scales as the search space grows. Each additional digit or variable can cause an explosive growth in required effort.

O(n)
Linear (Doubles with input)
O(n²)
Quadratic (Quadruples)
O(2ⁿ)
Exponential (Explodes)
O(n!)
Factorial (Infeasible quickly)

Password Cracking Simulations

Cybersecurity attackers use exhaustive search to break passwords. By taking a stolen cryptographic hash, they generate every possible password candidate, hash it, and compare it. The effectiveness of a password relies entirely on making the resulting search space impractically large.

Defense-in-Depth Mechanisms
Account Lockout ⊕
Locks the account after 3-5 failed attempts. Completely stops online brute-force attacks by limiting the search size directly.
Rate Limiting ⊕
Restricts guess attempts to 1 per second. Mathematically forces brute-force completion times into centuries.
Salted Hashing ⊕
Adds random data before hashing. Defeats pre-computed 'rainbow tables', forcing attackers to compute hashes individually for every user.
Multi-Factor Auth (MFA) ⊕
Requires a second physical token or app. Even if the password search space is successfully brute-forced, the attacker cannot login.

Network Vulnerability Scanning

Port scanning applies the brute-force concept to network reconnaissance. By attempting a connection (TCP Full Connect) to every single port sequentially, an attacker discovers listening services. Defenders rely on the predictable, noisy nature of brute-force to detect it.
