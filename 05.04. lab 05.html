📚 Beginner's Glossary

Every term from this module explained from scratch. Click a card to expand the full explanation.

🌐 Network Port
A numbered "door" (0–65,535) on a computer through which a specific type of network conversation flows.

Think of an IP address (Internet Protocol address) as a building's street address, and ports as numbered doors inside. Port 22 = SSH (Secure Shell), Port 80 = HTTP (HyperText Transfer Protocol), Port 443 = HTTPS (HTTP Secure). The official registry is maintained by IANA (Internet Assigned Numbers Authority).

Ports 1–1023 = well-known. Ports 1024–49151 = registered. Ports 49152–65535 = dynamic/private.
Show more ↓
🔍 Port Scanner
A program that probes a range of port numbers to discover which services are running on a machine.

A scanner attempts to connect to each port and records whether it is open (service listening), closed (nothing listening), or filtered (firewall blocks the probe). Sysadmins (system administrators) use scanners to audit their own infrastructure. The most widely cited open-source tool is Nmap (Network Mapper). In this exercise, you build your own.
Show more ↓
🐢 Throttling
Intentionally slowing an operation by inserting delays, limiting concurrency, or batching requests.

Without throttling, a port scanner fires thousands of connections per second, causing network congestion and triggering IDS (Intrusion Detection System) alerts. By inserting sleep() calls or limiting concurrent threads, the scanner becomes controllable. Same principle as API (Application Programming Interface) rate limiting, web scraping etiquette, and database connection pooling.

Big-O stays O(n) (linear — scales with the number of ports), but real wall-clock time grows proportionally with the delay.
Show more ↓
💥 Brute-Force Attack
Trying every possible password/key combination exhaustively until one works.

In algorithms, brute-force means systematic enumeration of all candidates. Applied to logins: try millions of passwords. A dictionary attack is a smarter variant using known common passwords. Credential stuffing uses leaked credentials from data breaches (tracked by Have I Been Pwned).

Complexity: O(A^L) (exponential — grows extremely fast with password length L over alphabet of size A). This is why long passwords matter. Your job: detect the attack, not perform it.
Show more ↓
📋 Security Event Logger
A component that records security-relevant events (logins, scans, alerts) to a persistent structured log.

Logs are foundational to SIEM (Security Information and Event Management) systems, forensic investigation, and compliance with regulations like GDPR (General Data Protection Regulation) and PCI DSS (Payment Card Industry Data Security Standard).

Each entry has: timestamp, severity level (INFO/WARNING/CRITICAL), source IP, event type, message. Write complexity: O(1) (constant — append to list). Query complexity depends on the data structure chosen.
Show more ↓
🪟 Sliding Window
An algorithm technique that maintains a moving range of recent data, discarding entries outside the window.

For brute-force detection: instead of counting all failures ever, count failures in the last 60 seconds. As time advances, the "window" slides forward and old events fall off the left edge. Implemented efficiently with a collections.deque (double-ended queue): append to right (new events), remove from left (old events). Amortized O(1) (constant amortized — each event is added and removed at most once) per event.
Show more ↓
🔌 TCP Socket
An endpoint for a TCP network connection, identified by (IP address, port) pair.

TCP (Transmission Control Protocol) is a reliable, ordered protocol. Connecting uses a three-way handshake: SYN → SYN-ACK → ACK. If a port is open, the server responds with SYN-ACK. If closed, it replies with RST (reset). If filtered (firewall), no response (timeout). Python's socket.connect_ex() returns 0 on success, non-zero on failure — perfect for scanning.
Show more ↓
🗺️ Attack Surface
The set of all entry points an attacker could potentially exploit on a system.

Every open port is a potential entry point. A sysadmin's goal is to minimize the attack surface: close ports for services that aren't needed, put firewalls in front of everything else, and monitor what remains. Port scanning your own system lets you see exactly what an outside attacker would see. Guidance from OWASP and NIST.
Show more ↓

def _probe_port(self, port):
    """Attempt TCP connection; return ScanResult with open/closed status."""
    t_start = time.time()
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.settimeout(self.timeout)
    result_code = sock.connect_ex((self.target, port))  # 0 = open
    sock.close()
    latency_ms = (time.time() - t_start) * 1000
    status  = "open" if result_code == 0 else "closed"
    service = KNOWN_SERVICES.get(port, "unknown")
    return ScanResult(port, status, service, latency_ms)

def scan_range(self, start_port, end_port, verbose=True):
    """Brute-force: try every port; sleep(throttle) between each."""
    for port in range(start_port, end_port + 1):
        result = self._probe_port(port)     # probe
        self.results.append(result)          # store
        if verbose and result.status == "open":
            print(result)                    # report
        time.sleep(self.throttle_delay)      # ← THROTTLE
    return self.results

Part 3 · ~45 min

🚨 Build: Brute-Force Attack Detector

Under normal use, a user might fail 2–3 times. An attack generates hundreds of failures per minute. Your detector finds this statistical anomaly using a sliding window over authentication events.

Sliding Window Algorithm

Pseudocode — Sliding Window Detection 
function process_event(event):
    if event.success: return None          # ignore successful logins

    window = failure_windows[event.ip]    # deque for this IP
    window.append(event.timestamp)        # add new failure

    cutoff = now() - window_seconds
    while window and window[0] < cutoff:   # evict old events
        window.popleft()                  # O(1) amortized

    if len(window) >= threshold:
        return fire_alert(event.ip)       # 🚨 brute-force detected!
    return None

Key Implementation: _evict_old_events()

Python · brute_force_detector.py 
def _evict_old_events(self, ip_address):
    """Remove failure timestamps older than window_seconds from deque front."""
    window = self.failure_windows[ip_address]  # collections.deque
    cutoff = time.time() - self.window_seconds
    while window and window[0] < cutoff:
        window.popleft()   # O(1) — deque left-removal is constant time
    # Each timestamp is added once and removed once → amortized O(1) total

Architecture

Architecture Diagram 
Incoming Event
      ↓  log()  [O(1) — constant append]
