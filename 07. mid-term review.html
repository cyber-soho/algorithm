1.2 Best, Expected, and Worst-Case Analysis

Every algorithm has three behavioral profiles. Best case (Ω): the minimum possible work. Worst case (O): the maximum possible work. Expected case (Θ): typical performance across all inputs. Capacity planning demands attention to worst-case behavior so servers don't fail under peak load.

Google SRE Book — engineering resource for reliability.

Chapter 2: Basic Sorting & Data Organization Module 02

2.1 The Three Sorting Champions

PropertyQuickSortMergeSortHeapSort

Best/Avg CaseO(n log n)O(n log n)O(n log n)
Worst CaseO(n²) — poor pivotO(n log n)O(n log n)
SpaceO(log n) in-placeO(n) extra arrayO(1) in-place
Stable?No (basic)YesNo

QuickSort selects a pivot, partitions into smaller/larger groups, and recurses. Worst-case O(n²) when pivot is always min/max. MergeSort always splits in half, merges in order — guaranteed O(n log n). HeapSort builds a max-heap, extracts the root repeatedly — O(1) space.

2.2 Sorting Network Traffic

Routers sort millions of packets per second for QoS (Quality of Service). Latency-sensitive traffic (video calls) gets priority over delay-tolerant data (downloads).

Cisco Systems · Cloudflare

2.3 Priority Queues

A priority queue serves elements by urgency, not arrival order. A binary heap gives O(log n) insert and O(log n) extract-max. Used in ER triage, OS scheduling, and cloud resource management.

AWS (Amazon Web Services)

Chapter 3: Binary Search Trees Module 03

3.1 BST (Binary Search Tree) Core Operations

The golden rule: for any node, all left-subtree values are smaller and all right-subtree values are larger.

OperationBalancedDegenerateProcess
SearchO(log n)O(n)Compare, go left or right
InsertionO(log n)O(n)Walk to empty spot
Deletion (two children)O(log n)O(n)Replace with in-order successor

3.2 Why Balance Matters

Sorted insertions (1, 2, 3, 4, 5) create a linked-list-shaped tree with O(n) operations. Balance keeps height at O(log n).

3.3 AVL (Adelson-Velsky and Landis) Trees

Self-balancing BST: subtree heights differ by at most 1. Rotations restore balance after insert/delete.

RotationTriggered WhenAction
LeftRight-Right casePivot unbalanced node down-left
RightLeft-Left casePivot unbalanced node down-right
Left-RightLeft child's right subtreeDouble rotation
Right-LeftRight child's left subtreeDouble rotation

4.2 Tail Call Optimization (TCO)

A tail-recursive call is the last operation with no remaining work. Compilers supporting TCO reuse the current frame, achieving O(1) memory.

Chapter 5: Brute-Force Algorithm Strategies Module 05

5.1 Exhaustive Search

Try every candidate, test each one. Guaranteed correct but trades efficiency for simplicity. Appropriate when the search space is small, correctness is critical, or as a baseline for verifying optimized solutions.
