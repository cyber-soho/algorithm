⚡ 2. Routing Algorithms & Complexity

This section compares the two primary Dynamic Programming (DP) algorithms explored in the lab: Dijkstra's Algorithm (single-source shortest path) and the Floyd-Warshall Algorithm (all-pairs shortest paths). The chart visualizes how computational time (operations) explodes as the network size grows, demonstrating why algorithmic choice is critical for large-scale infrastructure.

Dijkstra's Approach (Min-Heap)

A greedy/DP hybrid. It finds the shortest path from a single source by always visiting the unvisited node with the smallest known distance. Using a priority queue (min-heap), it achieves an efficient O((V + E) log V) time complexity.

Floyd-Warshall Approach (Tabulation)

Pure bottom-up DP. Computes shortest paths between ALL pairs simultaneously by testing every node as a potential intermediate stop. It builds a complete 2D matrix. Highly comprehensive, but computationally expensive at O(V³).

💾 3. Optimization & Caching Strategy

Finding the optimal route is computationally expensive. In real-world networks, queries are often repeated. This section demonstrates the impact of an LRU (Least Recently Used) Route Cache. As rounds of identical or overlapping queries are executed, the cache hit rate increases, bypassing the need to re-run routing algorithms entirely.

LRU Cache Mechanism

  • Stores recent query results in memory.
  • O(1) constant time lookup.
  • When full, evicts the least recently accessed item.
  • Utilizes a TTL (Time To Live) to prevent stale paths if topology changes.
